1. Shared (Ortak Kütüphane)
Shared projesi hem Producer hem de Consumer tarafından referans alınan ortak model ve fabrika sınıflarını barındırır. Docker build sırasında her iki Dockerfile da projenin kök dizinini context olarak alır, böylece Shared klasörüne erişebilir.
1.1 BaseLog.cs
Tüm log tiplerinin türediği soyut taban sınıftır. LogId (GUID), Timestamp, UserId, TCKN, Email, CreditCard ve LogLevel alanlarını içerir. GetLogType() soyut metodu her alt sınıf tarafından override edilmek zorundadır.
public abstract class BaseLog {
    public string LogId    { get; set; } = Guid.NewGuid().ToString();
    public DateTime Timestamp { get; set; } = DateTime.UtcNow;
    public string TCKN     { get; set; }
    public string Email    { get; set; }
    public string CreditCard { get; set; }
    public string LogLevel { get; set; } = "INFO"; // INFO|WARNING|ERROR|CRITICAL
    public abstract string GetLogType();
}
1.2 TransactionLog / ErrorLog / SecurityLog
BaseLog'dan türeyen üç somut log sınıfı borsa senaryolarını temsil eder:
• TransactionLog — IBAN, SWIFT, kaynak/hedef hesap bilgisi içerir.
• ErrorLog — ErrorMessage alanı ile sistem hatalarını tanımlar.
• SecurityLog — EventDetail ile güvenlik olaylarını (örn. yetkisiz erişim) tanımlar.
1.3 LogFactory.cs
Factory Pattern uygulamasıdır. CreateLog("TRANSACTION"|"ERROR"|"SECURITY") çağrısı ilgili somut sınıfın örneğini döndürür. Producer, hangi sınıfın somutlaştırıldığını bilmek zorunda kalmaz; bu sayede yeni log tipi eklemek yalnızca LogFactory'ye bir case eklemek anlamına gelir.
public static BaseLog CreateLog(string logType) =>
    logType.ToUpper() switch {
        "TRANSACTION" => new TransactionLog(),
        "ERROR"       => new ErrorLog(),
        "SECURITY"    => new SecurityLog(),
        _ => throw new ArgumentException($"Gecersiz log tipi: {logType}")
    };
2. Producer (Konteyner 1 — Veri Üretici)
Producer, borsa sistemini simüle eden ve sürekli log üreten modüldür. Docker ortamında CONSUMER_HOST ortam değişkeni ile Consumer'ın adresini alır (docker-compose'da "consumer-app" olarak tanımlı), yerel test için "127.0.0.1" kullanır.
2.1 Çalışma Akışı
1. Consumer'ın hazır olması için 2 saniye bekler.
2. TCP bağlantısı kurar (port 5000).
3. Sonsuz döngüde:
   a. LogFactory ile rastgele tip seçer (TRANSACTION/ERROR/SECURITY)
   b. Mock verilerle alanları doldurur (UserId, TCKN, Email, CreditCard)
   c. Rastgele LogLevel atar (INFO %~28, WARNING %~14, ERROR %~43, CRITICAL %~14)
   d. Nesneyi JSON'a serialize eder
   e. TCP stream'e yazar
   f. 500ms bekler (stres testinde bu gecikme kaldırılır)
3. Consumer — Pipeline (İşlem Zinciri)
Consumer'ın kalbi olan Pipeline, Chain of Responsibility tasarım kalıbıyla hayata geçirilmiştir. LogHandler soyut sınıfı zincirin temelini oluşturur.
3.1 LogHandler.cs — Zincir Temeli
SetNext(LogHandler) metodu ile zincirleme kurulur ve bir referans döndürür; bu sayede fluent syntax (a.SetNext(b).SetNext(c)) kullanılabilir. Her alt sınıf Handle(string) metodunu override ederek kendi işini yapar.
3.2 SeverityFilterHandler.cs — Adım 0
Zincirin ilk halkasıdır. Gelen JSON log string'i içinde "LogLevel":"INFO" veya "LogLevel":"WARNING" içeriyorsa return diyerek zinciri KESER — veri bir sonraki adıma geçmez. Yalnızca ERROR ve CRITICAL loglar devam eder. Bu gereksinim 3'ü (önemsiz logları filtreleme) karşılar.
bool isLowPriority =
    jsonLog.Contains("\"LogLevel\":\"INFO\"") ||
    jsonLog.Contains("\"LogLevel\":\"WARNING\"");
if (isLowPriority) {
    Console.WriteLine("[FILTRELEME] INFO/WARNING dusuruldu.");
    return; // zinciri kes
}
_nextHandler?.Handle(jsonLog);
3.3 KvkkFilterHandler.cs — Adım 1
KVKK (Kişisel Verilerin Korunması Kanunu) gereksinimlerini karşılar. Üç ayrı regex ile hassas verileri maskeler:
• TCKN (11 haneli sayı)  →  "***********"
• E-posta adresi          →  "***@***.***"
• Kredi kartı numarası    →  "****-****-****-****"
Maskeleme sonucu oluşan yeni string bir sonraki halkaya iletilir; orijinal hassas veri artık sistemde dolaşmaz.
3.4 EnrichmentHandler.cs — Adım 2
Log string'ine pipe-delimited (|) biçimde metadata etiketleri EKLER — yalnızca konsola yazmakla kalmaz, veriyi gerçekten zenginleştirir:
• sender_id   : Her işleme özgü rastgele UUID
• transaction_no : Zaman damgalı işlem numarası (TXN-yyyyMMddHHmmssfff)
• severity    : ERROR/Hata içeren loglar CRITICAL, diğerleri INFO
• message     : Kısa açıklama metni
• [DEBUG]     : Debug notu
3.5 FanOutHandler.cs — Adım 3
Tek bir logu birden fazla FormatterHandler'a "yayar". Ödev gereksinimi, üç farklı rolün farklı formatlarda çıktı almasını zorunlu kılar. FanOutHandler, foreach ile üç formattera sırayla aynı logu gönderir; her biri kendi stratejisiyle biçimlendirip PerformanceHandler'a iletir.
string[] roles = { "System Admin (HTML)", "CyberSec (CSV)", "Web Dev (JSON)" };
foreach (var formatter in _formatters) {
    formatter.SetNext(_sharedNext);
    formatter.Handle(jsonLog);
}
3.6 FormatterHandler.cs + Formatters — Adım 4
Strategy Pattern'ın uygulandığı noktadır. FormatterHandler bir IFormatterStrategy tutar; SetStrategy() ile dışarıdan atanır. Handle() çağrıldığında strateji üzerinden _strategy.Format(jsonLog) çağrısı yapılır:
Strateji	Hedef Rol	Çıktı Formatı
HtmlFormatter	System Admin	<div class="log-item"><p><b>Veri:</b> ...</p></div>
CsvFormatter	CyberSec	[CSV FORMATI] key:value, key:value, ...
JsonFormatter	Web Dev	[JSON FORMATI] { ham json string }
3.7 PerformanceHandler.cs — Adım 5
Zincirin son halkasıdır. İki görevi vardır:
1. Her başarılı logun işlendiğini raporlar.
2. Stres testi: %10 olasılıkla veriyi "drop" eder — yük altında backpressure mekanizmasını simüle etmek için.
Task.Delay kaldırılarak tam hızda çalıştırıldığında bu adım performans darboğazlarını görünür kılar.
4. Docker Yapılandırması
Her iki proje de çok aşamalı (multi-stage) build kullanan Dockerfile'a sahiptir:
• Aşama 1 (build): mcr.microsoft.com/dotnet/sdk:9.0 imajıyla derleme
• Aşama 2 (runtime): mcr.microsoft.com/dotnet/runtime:9.0 ile minimal imaj

docker-compose.yml dosyası:
• consumer-app önce başlar (producer depends_on ile bekler)
• CONSUMER_HOST=consumer-app ortam değişkeni ile Producer'ın adresi iletilir
• middleware-net bridge ağı ile konteynerler izole haberleşir
• Sistem performansı için her iki konteyner de resource limitleri ile yapılandırılır:
  - Memory Limit: 512MB (maximum)
  - Memory Reservation: 256MB (guaranteed)
  - CPU Limit: 0.5 cores
  - CPU Shares: 1024

docker-compose up --build    # iki konteyneri birlikte ayağa kaldırır
docker-compose down          # durdurur

5. OPTIMIZASYON DETAYLARI (Dönem Sonu Güncellemeleri)

5.1 Producer Optimizasyonu — Veri Üretim Hızı Artırılması
Dosya: Producer/Program.cs

İlk Durum:
  await Task.Delay(500);  # Saniyede 2 log

Yeni Durum:
  await Task.Delay(100);  # Saniyede 10 log (5x artış)

Amaç: Sistem performansını göstermek, yüksek throughput sırasında filtreleme ve maskeleme mekanizmalarını test etmek.
Sonuç: Producer artık saniyede 10 log üretir, bu da Consumer'ın işleme kapasitesini test etmeye olanak tanır.

5.2 Consumer Optimizasyonu — Network Buffer Artırılması
Dosya: Consumer/Program.cs

İlk Durum:
  byte[] buffer = new byte[4096];  # 4 KB buffer

Yeni Durum:
  byte[] buffer = new byte[65536]; # 64 KB buffer (16x artış)

Amaç: Daha fazla veriyi tek seferde ağdan okuyarak throughput'ı artırmak.
Sonuç: Network I/O performansı ciddi şekilde iyileşti.

5.3 Consumer Error Handling Iyileştirmesi
HandleClientAsync metoduna try-catch-finally bloğu eklendi:
  try {
      while ((bytesRead = await stream.ReadAsync(...)) != 0) { ... }
  } catch (Exception ex) {
      Console.WriteLine($"[HATA] Veri işleme sırasında hata: {ex.Message}");
  } finally {
      client.Close();
  }

Amaç: Güvenilir bağlantı yönetimi, hata durumunda da client'ı temiz kapatmak.

5.4 Docker Resource Limits
Her iki konteyner için resource limits eklendi:
  consumer-app & producer-app:
    mem_limit: 512m              # Maksimum bellek
    mem_reservation: 256m        # Minimum garantili bellek
    cpus: '0.5'                  # Maksimum CPU
    cpu_shares: 1024             # CPU priority

Amaç: Container'ların sınırsız kaynak tüketmesini engellemek, production ortamında stabil çalışma sağlamak.
